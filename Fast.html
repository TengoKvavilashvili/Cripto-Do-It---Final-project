A major reason why vectorization is faster than its for loop counterpart is due to the underlying implementation of Numpy operations. As many of you know (if you’re familiar with Python), Python is a dynamically typed language. When you declare a variable, no datatype needs to be specified since Python will infer it for you during runtime. This is different from a statically typed language which requires datatype to be specified in order for the compiler to check, and these languages include Java and C. In Python, a list is assumed to store heterogeneous elements, and during runtime, each individual element in the list has to be checked (type check) when performing a certain operation. This will add certain overhead to a regular list. On the other hand, Numpy uses ndarray, which is multi-dimensional array of fixed-size items storing homogeneous elements. ndarray is optimized as the underlying operations are done using efficient C operations, which allows for vectorization.
